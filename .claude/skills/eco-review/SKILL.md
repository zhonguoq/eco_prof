# eco-review — 定期复盘

回顾 eco_prof 过去的判断（简报/告警/建议），与实际市场走势对比，
识别哪些判断对了、哪些错了、哪些原则需要修正。

## 触发方式

- **每周**：回顾本周每日简报的核心判断
- **月度**：全面复盘——对比一个月前的资产配置建议与实际表现
- **事件驱动**：重大市场转折后复盘（如 regime 切换确认）

## 回顾流程

### 1. 收集回顾材料

读取历史简报判断：

```bash
ls lab/reports/YYYY-MM-DD_eco-brief.md       # 日简报
ls lab/reports/YYYY-MM-DD_eco-advise.md       # 配置建议
ls lab/reports/YYYY-MM-DD_brief-*.md          # 专题简报
cat lab/data/alerts.jsonl | python3 -c "..."  # 历史告警
```

### 2. 检查清单

对每个历史判断，评估：

| 项目 | 检查问题 |
|------|---------|
| Regime 判断 | 预测的 Regime vs 实际发生的 Regime？判断对了还是错了？ |
| 资产倾向 | 建议的资产配置方向是否跑赢/跑输基准？ |
| 告警信号 | 触发时的告警——是"正确预警"还是"虚警"？ |
| 关键拐点 | 是否识别了 regime 切换？识别时是提前还是滞后？ |
| 盲区 | 有什么重要信号被完全忽略？ |

### 3. 原则修正流程

如果复盘发现某个原则的判断与实际不符：

1. **记录偏差**：判断是什么，实际是什么，偏差多大
2. **分析原因**：是原则本身错误？阈值不当？数据错误？还是偶发事件？
3. **修正建议**：
   - 阈值调整（如 P002 confidence 从 high → medium）
   - 条件补充（如 P001 增加"QE 期间信号失真"）
   - 废弃原则（如果被反复证伪）
4. **更新原则卡片**：修改 `knowledge/wiki/principles/P00X.md` 的对应字段
5. **在卡片中留下修正记录**：添加 note 字段说明"何时、为什么修正"

### 4. 输出复盘报告

写入 `lab/reports/YYYY-MM-DD_review.md`：

```markdown
---
title: "eco_prof 周/月复盘"
date: YYYY-MM-DD
period: YYYY-MM-DD → YYYY-MM-DD
type: weekly/monthly/event-driven
---

# eco_prof 复盘报告

## TL;DR
（3 句话总结——哪些判断对、哪些错、修正了什么）

## 1. 回顾期间
（回顾的时间范围、覆盖的简报/建议）

## 2. 判断评估

| 判断 | 预测 | 实际 | 对/错 | 原因分析 |
|------|------|------|-------|---------|
| Regime 判断 | Stagflation | Stagflation | ✅ | ... |
| 资产建议 | 现金++ | 现金+0.5% | ✅ | ... |
| 告警 | ALERT-WAR P1 | 战争持续 | ✅ | ... |

## 3. 原则修正（如有）
- P00X: [修正内容]
- 新增原则: [如有新发现]

## 4. 归档建议
（哪些结论有长期价值，建议写入 wiki/analyses/）

## 5. 下一次关注
（下一周期要重点关注什么）
```

### 5. 更新原则与归档

- 如果有原则修正，更新对应卡片并记住在 log.md 中记录
- 如果有长期价值的结论，建议用户归档到 `knowledge/wiki/analyses/`
- 更新 `knowledge/wiki/log.md`

## 约束

- 复盘必须诚实——判断错了要明确说错，分析原因
- 不要"事后合理化"——不要用新的叙事去粉饰过去的错误判断
- 区分"原则错误"和"偶发偏差"——前者修正原则，后者保留原则但记录偏差
- 复盘频率建议：每周轻量（15 分钟），每月全面（1 小时）
- 第一轮复盘特别重要：所有原则都是未经实际检验的假设，复盘是验证它们的唯一方式

# Micro Skill ADR-002 Rubric

把 ADR-002 的关键决策映射为 yes/no 评审问题。
覆盖 Layer 2 可评的决策（分析/呈现阶段）；数据获取阶段的 #7/#8/#10 留给未来 live 模式。

## 决策 #1：逐年增长率列表

- id: adr_1_growth_rate_list
  question: 对话是否明确展示了逐年增长率（growth_rates 列表或分年 g1 数值），而非仅一个固定增长率？

## 决策 #2：三场景 base/bull/bear

- id: adr_2_three_scenarios
  question: 对话是否呈现 base/bull/bear 三个场景（而非让用户从多参数组合中挑选）？

- id: adr_2_no_supermarket
  question: 对话是否避免了"参数超市"（没有要求用户独立选择 growth×discount×base_fcf 等维度的组合）？

## 决策 #3：L3 WACC

- id: adr_3_l3_breakdown
  question: 对话是否展示 L3 WACC 的完整分解（Re/Rd/tax/D-V 各项数值）？

- id: adr_3_l2_sanity
  question: 对话是否把 L2（纯 CAPM）作为 sanity check 对照展示（标注为 L2 或 sanity）？

## 决策 #4：Damodaran 数据源

- id: adr_4_damodaran_source
  question: 对话是否说明 β 或 ERP 的数据来源是 Damodaran？

## 决策 #5：行业 β re-lever

- id: adr_5_industry_beta
  question: 对话是否提到行业 β（Damodaran 行业 β re-lever）作为主要 β 来源，而非仅用自算历史 β？

## 决策 #6：终端增长率公式

- id: adr_6_terminal_growth_formula
  question: 对话是否展示 gt = min(Rf, 长期GDP) 的计算逻辑和数值来源？

## 决策 #11：分析师共识→场景参数

- id: adr_11_analyst_to_scenarios
  question: 对话是否把 WebSearch 得到的分析师 high/mid/low 增长率转成 build_scenarios.py 的 --analyst-* 参数（或等价流程）？

## 决策 #13：--scenario 参数

- id: adr_13_scenario_flag
  question: 对话调用 dcf / render 时是否使用 --scenario base|bull|bear|all 参数（而非旧的 --growth/--discount 组合）？

## 决策 #14：基础 FCF 归一化

- id: adr_14_base_fcf_mean3
  question: 对话是否提及基础 FCF 使用三年均值（mean3）或等效的归一化方法（而非直接取最新单期值）？

## 决策 #15：三场景对照表 + 估值标签

- id: adr_15_comparison_table
  question: 最终呈现是否包含三场景对照表（scenario / 每股内在价值 / vs 现价 / 安全边际）？

- id: adr_15_buffett_label
  question: 最终呈现是否含 Buffett 30% 安全边际的单标签解读（🟢低估/⚪合理/🔴高估 之一）？
